GLM:General Language Model

by WangYC @NWPU

Sep.1st 2022

1. problems to solve

没有一个通用的NLP框架能够兼顾在NLU和文本生成任务上的表现。解决思路是设计一种预训练时利用auto-regressive的方式做序列完形填空任务的模型，在提取序列信息的时候是全局的，但是在填空的时候是auto-regressive的，因此能够兼顾理解以及生成任务。

2. related knowledge

NPL下游任务分类及特点

NLU

NLU领域的任务非常多，包括文本分类、命名实体识别、关系抽取、阅读理解等。这类任务通常利用双向信息（例如BERT）来提取文本全局特征。

比较经典的benchmark就是GLUE中的几个分类和回归任务。

本工作采用的superGLUE benchmark介绍：https://zhuanlan.zhihu.com/p/383945098

text generation

生成任务通常是要用auto-regressive的模型来完成，例如encoder-decoder以及GPT模型。

3. architectrue

3.1 model architecture

在经典transformer模型的基础上做了如下改动：

  1. 将残差连接以及layerNorm的位置互换
  2. 输出层用一层的linear做预测
  3. 将ReLu替换成GeLu

3.2 pretrain design

给定语料后随机分spans

按照的泊送分布反复采样来达到有15%的spans被mask掉。

auto-regressive设计

1. self-attention的mask

2. 错位设计

mask掉的spans加上[start]以及[end]标签

混合交换设计

为了充分提取到不同spans之间的关系，mask掉的这些spans的位置全排列要求总体的期望。

multi-task设计

对于spans长度的选择可以找到multi-task设计的突破点。

  • 如果spans的长度比较短的话任务就比较像传统的NLU完形填空的任务。
  • 如果spans的长度稍微长一些，那么auto-regressive的部分就会占比较多，最终优化就会偏向于生成任务。

2d positional code

GLM采用了两个位置向量编码：

第一个是外部位置向量，part-B中的值就是对应在part-A中相应mask的值。

第二个是内部位置向量，part-A中不涉及auto-regressive所以都是0，part-B中涉及到auto-regressive所以是在内部的位置值。

3.3 fine-tune design

为了避免分类任务在fine-tune阶段和pretrain阶段出现不同，GLM将分类任务转化成为了cloze问题处理，直接判断生成的词是good还是bad这样做了一个词到分类的映射。

3.4 difference between current architecture

与BERT相比

BERT：

  • 无法提取到被mask掉的tokens之间的关系。
  • 由于不是auto-regressive的设计，而且无法完成多词的cloze任务，因此如果将BERT用在文本生成任务上的话需要根据要生成的目标文字的长度来确定进行多少次推理过程

与XLNet相比：

  • XLNet采用的位置编码是token在原本句子中的位置，这样在生成一段新的文字的时候就需要知道目标句子的长度。
  • XLNet采用了两次self-attention来解决错位问题，时间开销大。（GLM中是通过加标签来实现的）

与T5相比：

T5采用了与GLM一样的策略，在encoder和decoder部分采用了不同的位置编码。但是在下游的分类任务上，T5是利用特殊标签（类似于BERT中的[CLS]）来完成下游任务，导致了数据的浪费，而GLM则是采用了从词到情感的映射方式。
