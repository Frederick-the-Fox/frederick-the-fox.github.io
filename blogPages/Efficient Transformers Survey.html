1. basic knowledge of transformer

下面从经典transformer结构、时间复杂度、transformer的mode三个角度进行介绍。

1.1 architecture

Transformer是多个基本块堆叠而成的多层结构。

1.1.1 transformer blocks

基本块由multi-head attention, point-wise feed-forward, layerNorm以及残差连接层构成：

1.1.2 attention

attention，注意力机制，最常用的为additive attention（允许Q和K的长度不相同）和 dot-product attention（经典transformer采用的就是dot-product）。

attention由Q, K, V三个矩阵的相乘运算完成从输入到输出的过程。

1.1.2.1 单头attention过程

值得注意的是，这个过程中隐含的约束：

  1. key与value的数量相同
  2. query与key的维度相同
1.1.2.2 multi-head attention:

由于dot-product attention本身并没有可计算的参数，因此采用multi-head的方式来增加模型的学习能力。

将原本的维度投影到低维度，分别进行单头attention计算，最后concat到一起。

在经典transformer中采用的多头注意力机制，head的数量h = 8。

1.1.3 position-wise feed-forward layers

position-wise 也被叫作 point-wise ，指的是对序列中的每一个token在最后一个维度做线性展开。

经典transformer的embedding空间为512，其FF做法是512到64再到512 。

1.1.4 mask

在例如机器翻译的应用场景中，要考虑时间顺序，在做第k个token的加权时不应该出现k以后的token的贡献。

这也被称为是causal的。

1.2 Efficency

attention的事件复杂度和空间复杂度均为， 这妨碍模型在多种环境下的可扩展性。

时空复杂度：https://zhuanlan.zhihu.com/p/264749298

最近出现了大量试图解决该问题的 Transformer 模型变体，本文将这类模型称作「高效 Transformer」(efficient Transformer)。

1.2.1 defination of efficiency

自注意力机制是 Transformer 模型的核心典型特征。对于自注意力的一个担忧是其时空复杂度都是平方级的，这妨碍模型在多种环境下的可扩展性。

基于此，模型的效率有了不同的诠释。效率可能指模型的 内存占用，当模型运行的加速器内存有限时这尤为重要；效率也可能指训练和推断过程中的 计算成本，如 FLOPs 数。尤其对于设备端应用而言，模型应在有限的计算预算下运行。该综述论文从 内存 和 计算 两个角度来考虑 Transformer 的效率。

1.3 mode of transformers

Transformer模型根据使用的方式不同可以主要分为三个mode：

  1. encoder-only（例如：分类任务）
  2. decoder-only（例如：语言模型）
  3. encoder-decoder（例如：机器翻译）

对于encoder-decoder mode，一般包含多个multi-headed self-attention模块，其中包括encoder和decoder中的标准self-attention模块以及一个encoder-decoder的cross-attention让decoder来利用encoder的信息。这影响了self-attention机制的设计。

对于encoder mode，对self-attention机制是否需要是causal的是没有限制的，即完全依赖于过去和现在的token。

对于encoder-decoder的设置，encoder和encoder-decoder的cross attention可以是non-causal的，但是decoder的self-attention必须是causal的。

支持causal auto-regressive decoding的能力是设计高效self-attention机制是需要的，因为这是在许多应用中的一个限制因素。

 

2. efficient X-formers

Efficient Transformers: A Survey这篇综述聚焦于通过解决自注意力机制的平方级复杂度来提升 Transformer 效率的建模发展和架构创新，同时该论文还简要探讨了通用改进和其他效率改进。

2.1 ⭐️ A Taxonomy

本部分将高效Transformer进行分类。除了基于segment-based recurrence模型外大部分模型的主要目标是近似二次cost的attention矩阵（approximate the quadratic-cost attention matrix）。每一个方法将一些稀疏的方法用于原本稠密的attention机制。

2.1.1 Fixed Patterns(FP)

最早对于self-attention的改动就是简单的通过限制field of view为固定值，预定义patterns（例如local windows以及固定步长的block patterns）对attention矩阵进行稀疏化。

通过限制attention的范围——从全局变成局部——降低计算复杂度。限制范围的方法包括blockwise pattern，strided pattern，compressed pattern。

Blockwise pattern 将输入序列切成多个block，attention只发生在block范围内，所以复杂度从降到，B为block size，但是这样切割会导致序列不连贯，attention能力受限，效果应该不太行。

Strided pattern 则采用滑动窗口的形式，每个token与周围相邻的几个token作attention，相邻的token范围就是window size，这样是有一定道理的，因为自然语言在多数情况下都是局部相关的，所以在一个窗口范围内作attention往往不会丢失太多信息，相比之前的blockwise pattern应该好一些。

Compressed pattern 则是先通过卷积池化对序列进行down sampling，比如用核为2，步长为2的CNN，将2个token表征成一个向量，然后在做attention，同样也能降低attention的计算复杂度。其实就相当于通过CNN对序列进行n-gram的切分。

2.1.2 Learnable Patterns(LP)

learnable patterns是对上文提到的fixed patterns的扩展，简单来说fixed pattern是认为规定好一些区域，让该区域的token进行注意力计算，而learnable patterns则是通过引入可学习参数，让模型自己找到划分区域。比如reformer引入基于哈希的相似度度量方法将输入序列切割；routing transformer对token向量进行k-means聚类来将整体序列分割成多个子序列。所以本质上说LP与FP是一致的，都是通过将整体序列 切分成子序列，attention只在子序列中进行，从而降低计算开销，只不过LP的区域划分是通过模型学得，而FP则是人为定义。

2.1.3 Memory

memory乍一听好像有点让人摸不着头脑，其实想法也很简单。最开始是在19年的set transformer上使用。一般来说做multihead self-attention时，Q=K=V=X（X为输入序列，长度为n），而在set transformer中，作者先单独设置了m个向量（m是超参数），然后这m个向量与X做multihead attention，得到m个临时向量（这些个临时变量我们就称作“temporary memory”），接着把X与这m个临时向量再做一次multihead attention得到输出。

这个过程其实就是利用这m个向量将输入序列X的信息先通过attention进行压缩，再通过attention还原，达到抽取输入序列的特征的目的。但是在压缩编码解码的过程中肯定会有信息损失，所以后来的改进方法是引入全局记忆，即设置一些token，它们可以与所有的token进行注意力交互，比如bert的[CLS]，由于这些token的数目远小于序列长度，因此也不会给计算带来负担，而且往往携带了整个输入序列的信息，这就是可以在bert上用[CLS]作文本分类任务的原因。

2.1.5 Kernels

另一种近期比较火的方法是去通过kernelization来view attention机制，以核函数变换的新形式取代原有的softmax注意力矩阵计算。 kernel方法可以更巧妙的对self-attention机制进行数学重写，来避免对矩阵显式计算，将计算复杂度降至 范围内。因为核方法是对attention矩阵的近似，因此其也可以看做low-rank方法的一种。比较代表的有Linear Transformers

2.1.6 Recurrence

recurrence实际上也是上文提到的fixed patterns中blockwise的一种延伸。本质上仍是对输入序列进行区域划分，不过它进一步的对划分后的block做了一层循环连接，通过这样的层级关系就可以把一个长序列的输入更好的表征。以recurrence为代表的就是Transformer-XL。

2.2.2 Longformer

longformer初衷为解决长序列缩短会丢失重要信息的问题。它的attention包括窗口化的局部上下文的self attention和由终端任务激活的全局attention。

局部attention用来建立局部的上下文表示，全局attention用来建立完整的序列表示以进行预测。

longformer主要采用了滑动窗口+全局token来实现self-attention的稀疏计算。其中滑动窗口就是每个token只与其左右 范围内的token发生注意力，另外弥补滑动窗口对更长文本注意力的不足，作者参考空洞卷积网络引入了空洞滑动窗口，旨在把相距更远的token相关联。在multihead计算中，同时引入sliding window与dilated window到不同的head中，可以提升效果。除了滑动窗口，加入一些全局token也会提高模型的能力。

对于一些任务，比如说QA，需要通过question去找document里的答案，因此局部注意力在这种情况下就不适用。因此，作者就在一些预先选择的位置上添加了全局注意力。在这些位置上，会对整个序列做attention。

滑动窗口将attention的矩阵乘法计算复杂度以及参数memory缩小到了，其中是滑动窗口的大小。

2.2.3 Reformer

reformer采用了局部敏感哈希来降低self-attention的计算量。我们知道所谓的attention score实际上就是向量之间的内积，如果两个向量在特征空间上邻近，其内积就会很大，两个向量就会越相关，注意力得分也会越高；相反离得很远的向量内积也会很小，就越不相关。而我们关注的往往是注意力得分高的向量，所以计算注意力矩阵的问题就变成了寻找近邻向量的问题。怎样快速找到高维空间的近邻向量呢？一种方法就是局部敏感哈希技术（LSH）。首先，引入一个固定的随机矩阵R，维度为 ， 为模型特征维度，为哈希数，表示总共需要多少个hash值，是一个超参数，b设的越大，hash值越多。对一个输入序列x（维度 n×dmodel ），每一个token向量经过hash函数后就能得到一个确定的hash值，如果两个token的hash值一样，那么这两个token向量就是近邻向量，它们之间就可以计算self-attention。

hash的过程类似于将一个转盘随机翻转b次，看每次分别落在哪几个区域中

3. more discussion

关于efficient transformers的一些规律以及仍然可以探索的问题：

3.1 关于evaluation

目前X-formers通常自己选取benchmarks来展示自己的模型效果，同时由于预训练以及是否支持causal等因素导致目前很难有一个比较统一的evaluate各模型的efficiency的方法。有一些工作在探索统一的评估方法：Long Range Arena, i.e., LRA, (Tay et al., 2021a)（Ps:还是一作自己的工作）

3.2 其他方法

最早的尝试是fixed pattern，后来到learnable的pattern，再后来有稀疏性以及memory机制的出现，

  • 参数共享
  • 量化 / 混合精度
  • 知识蒸馏
  • 神经架构搜索
  • Task Adapter

3.3 future diretions

  1. 在保证高效的同时保证通用性
  2. 不应该以内存换速度，不应该牺牲TPU的封装能力
  3. 或许不应该是一种变体，而是一种全新的思路（终极的x-former还是不是transformer）
